# AOM (Active Object Model)
    该库是一个 Header-only（仅头文件）的现代 C++ 库，非常适合直接集成
    https://samiralavi.github.io/activecpp/main/

    1. 核心设计理念
    AOM 通常围绕以下几个关键组件构建，将“并发编程”转化为“消息传递”：
    代理 (Proxy)：对外暴露的接口。客户端像调用普通对象方法一样调用代理，但实际上它只是将请求封装成了一个任务。
    调度器 (Scheduler/Servant)：内部维护一个任务队列。它按照特定的策略（通常是 FIFO）从队列中取出任务并在自己的私有线程中执行。
    Future/Promise：处理异步结果。调用者不会被阻塞，而是拿到一个“凭证”，等执行完成后再获取结果。
    AOM 的主要优势
    线程安全自然化：因为每个 Active Object 都在自己的线程中运行，对象内部的状态不需要加锁（除非涉及跨对象交互），有效避免了繁琐的 std::mutex。
    降低耦合：调用方不需要关心任务是在哪个线程执行的，也不需要管理线程的生命周期。
    提高响应性：主线程（如 UI 线程）可以将耗时操作丢给 Active Object，从而保持界面的流畅。
